# bachata.co.kr 제품 기획서

작성일: 2026-07-09  
제품명: 바차타 코리아 / Bachata Korea  
방향: 토픽 기반 한국형 바차타 커뮤니티, 레딧의 쓰레드 구조와 팀블라인드의 주제 탐색 UX를 결합한 모바일 우선 웹앱

## 1. 한 줄 정의

바차타 코리아는 바차타 영상, 행사, 댄서, 수업 후기, 소셜 후기를 토픽으로 엮고, 댓글과 추천으로 읽을 만한 글이 오래 살아남게 만드는 한국형 바차타 커뮤니티입니다.

## 2. 제품 미션

한국 바차타 씬은 정보가 인스타그램, 유튜브, 네이버/다음 카페, 블로그, 오픈채팅, 학원 공지, 개인 계정에 흩어져 있습니다. 사용자는 처음 배울 때 어디서 시작해야 하는지, 어떤 행사가 가까운지, 어떤 댄서의 영상을 보면 좋은지, 특정 수업이나 소셜 분위기가 어떤지 한 번에 파악하기 어렵습니다.

바차타 코리아의 미션은 세 가지입니다.

1. 바차타 관련 정보를 토픽 단위로 모아 검색과 탐색이 쉬운 구조를 만든다.
2. 비회원도 가볍게 글과 댓글을 남길 수 있게 하되, 임시비밀번호와 IP 일부 표시로 최소한의 책임성을 둔다.
3. 영상, 행사, 댄서, 수업 후기가 댓글 토론과 연결되도록 만들어 커뮤니티 자체가 최신 정보 아카이브가 되게 한다.

## 3. 핵심 포지셔닝

레딧처럼:

- 글 하나가 독립된 쓰레드가 된다.
- 추천/비추천과 댓글로 좋은 글이 올라온다.
- 영상, 링크, 이미지, 텍스트, 설문, 무물보를 같은 작성 흐름에서 다룬다.
- 각 상세 페이지에는 본문 바로 아래 추천, 비추천, 댓글, 공유 액션이 보인다.

팀블라인드처럼:

- 홈은 토픽 베스트 중심으로 간다.
- 상단 대카테고리는 `토픽 베스트`, `주제 탐색`, `아카데미 리뷰`, `댄서 리뷰`로 단순하게 유지한다.
- `주제 탐색`은 채널 탐색처럼 주제 카드와 하위 주제를 보여준다.
- 아카데미 리뷰와 댄서 리뷰도 별도 게시판이 아니라 해당 카테고리의 토픽 피드다.
- 사용자는 관심 주제를 팔로우하거나, 주제별 글을 추천순/최신순으로 볼 수 있다.

한국형 커뮤니티처럼:

- 비회원 글쓰기와 댓글을 허용한다.
- 비회원은 랜덤 닉네임 또는 직접 닉네임을 쓴다.
- 작성자는 4자리 임시비밀번호를 직접 정한다.
- 비회원 IP는 앞 두 옥텟만 공개하고 나머지는 숨긴다.
- 댓글과 글은 가볍지만, 신고/삭제/관리 여지를 남긴다.

## 4. 사용자

### 입문자

궁금한 것:

- 바차타 베이직이 무엇인지
- 센슈얼과 도미니칸이 어떻게 다른지
- 첫 소셜에 가도 되는지
- 어느 학원이나 동호회가 초보자에게 편한지

필요 기능:

- 질문 토픽

- 댓글로 바로 물어보기

- 이번 주 어디서 소셜이 열리는지

필요 기능:

- 무물보

## 6. 핵심 기능

### 6.1 토픽 피드

목표:

- 홈은 토픽만 계속 이어지는 피드여야 한다.
- 주제 탐색은 별도 페이지로 분리한다.
- 카테고리 페이지도 결국 토픽 리스트다.

정렬:

- 추천순
- 최신순
- 댓글 많은 순
- 상승 중

현재 구현:

- 정적 seed 토픽과 D1 guest thread를 함께 보여준다.
- `LiveThreadList`가 D1 글을 가져온다.
- 홈은 `FeedTabs`, `BestContentRail`, `LiveThreadList`, seed thread list로 구성된다.

주의:

- 목업 추천수와 목업 댓글 수는 공개되면 안 된다.
- 추천/비추천 수는 실제 API 값만 보여야 한다.

### 6.2 글쓰기

목표:

- 비회원도 글을 쓸 수 있다.
- 작성자는 닉네임과 4자리 임시비밀번호를 직접 정한다.
- 제목에 자동 예시 문구를 넣지 않는다.
- 글 유형은 텍스트, 이미지 및 동영상, 링크, 설문조사, 무물보를 지원한다.
- 주제와 하위 주제를 선택한다.

현재 구현:

- `/write`에서 비회원 글쓰기를 지원한다.
- 제목 placeholder는 `제목을 입력하세요`다.
- 이미지/동영상 탭에서 파일 업로드 UI를 표시한다.
- 파일은 Cloudflare R2 `bachata-media`에 업로드한다.
- D1 `guest_threads`에 글을 저장한다.

다음 보강:

- 첨부 파일 업로드 진행률 표시.
- 업로드 실패 시 파일별 에러 표시.
- 게시 전 미리보기.
- 글 수정/삭제.
- 신고 기능.

### 6.3 비회원 댓글

목표:

- 댓글 영역은 처음에는 작은 입력창으로 보인다.
- 입력창을 터치하면 닉네임, 임시비밀번호, 본문, 등록 버튼이 펼쳐진다.
- 댓글 등록 버튼은 항상 명확하게 보여야 한다.
- 댓글에도 추천/비추천이 있다.
- 댓글 정렬은 좋아요 비율 높은 순, 최신순, 오래된순을 지원한다.

현재 구현:

- `LiveComments`가 댓글 입력, 등록, 정렬, 검색, 답글, 댓글 추천/비추천을 지원한다.
- 댓글은 D1 `comments`에 저장된다.
- 댓글 추천은 D1 `comment_votes`에 저장된다.

주의:

- 임시비밀번호는 자동 입력하면 안 된다.
- 등록 후 비밀번호는 비워야 한다.
- 등록 완료 문구가 입력창 안에 끼어 보이면 안 된다.

### 6.4 익명 닉네임

목표:

- 기본 닉네임은 실제 활동명처럼 보이지 않아야 한다.
- 안전한 바차타 용어 + 3자리 숫자로 만든다.
- 예: `센슈얼218`, `도미니칸881`, `인플루언스021`, `풋워크661`, `라떼랄504`, `바일라730`

현재 구현:

- 랜덤 닉네임 함수는 존재한다.

다음 보강:

- 실제 활동명과 혼동될 수 있는 조합 금지.
- 접속 세션 동안 같은 닉네임 유지.
- 같은 닉네임 중복 가능성을 낮추기 위한 숫자 범위와 세션 키 정비.

### 6.5 추천/비추천

목표:

- 모든 토픽 카드와 상세 페이지에 추천/비추천이 보여야 한다.
- 영어 `Like`, `Dislike`가 아니라 화살표 아이콘으로 간다.
- 클릭하면 실제 D1 API에 반영된다.
- 정적 seed 글도 투표 API 대상이 된다.

현재 구현:

- `VoteRail`이 `app/api/votes`와 연결된다.
- `thread_vote_totals`, `thread_votes` 테이블을 사용한다.
- 정적 글, 비회원 글, 댄서 페이지, 행사 페이지에 투표 타깃이 붙어 있다.

주의:

- 서버에 저장된 값이 없는 경우 seed 기본값을 그대로 보여준다.
- seed 기본값은 목업 숫자가 아니라 0이어야 한다.

### 6.6 공유 미리보기

목표:

- 카카오톡, 문자, 커뮤니티에 링크를 붙였을 때 제목, 설명, 썸네일이 뜬다.
- 영상이 있는 글은 유튜브 썸네일을 쓴다.
- 글만 있는 토픽도 기본 OG 이미지를 가져야 한다.
- 가능하면 베스트 댓글 일부나 댓글 수를 설명문에 반영한다.

현재 구현:

- 정적 상세 페이지에 OG/Twitter 메타를 넣고 있다.
- 영상 있는 글은 유튜브 썸네일을 사용한다.

다음 보강:

- 글만 있는 토픽용 기본 OG 이미지 생성.
- D1 guest thread 상세의 동적 OG 메타 보강.
- 카카오톡 미리보기 실제 테스트 루틴 문서화.

### 6.7 행사 페이지

목표:

- 행사 카드의 썸네일을 누르면 바차타 코리아 내부 행사 상세로 들어간다.
- 내부 행사 상세에는 포스터, 일정, 위치, 공식 링크, 설명, 댓글이 있어야 한다.
- 공식 링크는 외부 확인용이고, 커뮤니티 대화는 내부 쓰레드에서 이어진다.

현재 구현:

- `/events/[id]` 상세 페이지가 있다.
- K-Sensual, Seoul Bachata Festival, Bachata Geneva Festival 상세가 생성된다.
- 댓글과 추천/비추천이 붙어 있다.

다음 보강:

- 실제 행사 포스터 수집/저장.
- 날짜/장소/패스 범위 검증 상태를 운영용으로만 관리.
- 행사 후기 토픽 자동 연결.

### 6.8 댄서 페이지와 댄서 리뷰

목표:

- 댄서 소개는 단순 프로필이 아니라 댓글이 붙은 토픽형 상세여야 한다.
- 댄서 리뷰는 수업/워크숍/소셜댄스 경험을 남기는 피드다.

현재 구현:

- `/dancers/[id]` 상세에 영상, 태그, 추천/댓글/공유가 붙어 있다.
- `/topics/dancer-review`가 있다.

다음 보강:

- 더 많은 국내외 댄서 데이터 추가.
- 내한 기록, 공개 영상, 공식 링크 근거 분리.
- 댄서 리뷰 작성 시 참여 유형 선택 필드 추가.

### 6.9 로그인과 프로필

목표:

- 구글 로그인 기반 회원가입을 제공한다.
- 회원은 닉네임, 프사, 프로필을 수정할 수 있다.
- 비회원은 글/댓글을 쓰되 임시비밀번호 기반으로 수정/삭제한다.

현재 구현:

- Supabase Google OAuth 버튼 컴포넌트와 `/login` 페이지가 있다.

남은 작업:

- Supabase URL과 anon key 환경변수 연결.
- Google Provider 설정.
- 로그인 콜백 처리.
- 회원 프로필 저장 테이블.
- 회원 작성글/댓글 권한 체크.

### 6.10 미디어 업로드

목표:

- 글쓰기에서 이미지와 동영상을 올릴 수 있다.
- 작은 용량으로 최적화한다.
- 장기적으로 기본 랜덤 프사와 춤 관련 에셋도 제공한다.

현재 구현:

- Cloudflare R2 `bachata-media` 버킷 연결.
- `/api/uploads`와 `/api/media/[...key]`가 있다.
- 이미지와 일부 동영상 MIME을 허용한다.

다음 보강:

- 업로드 용량 정책 명확화.
- 썸네일 생성.
- 영상 직접 업로드 비용/트래픽 제한.
- 악성 파일 방어.

### 6.11 어워드

목표:

- 레딧식 장식 기능을 바차타스럽게 귀엽게 만든다.
- 예: 풋워크 좋음, 음악성 좋음, 매너 좋음, 플로우 좋음.

현재 구현:

- `AwardPicker`와 `/api/awards`가 있다.

검토:

- 초기에는 상세 본문 하단의 보조 반응으로 두고, 과하게 노출하지 않는다.

## 9. UI/UX 기준

### 모바일 우선

- 독자가 볼 필요 없는 내부 문구.

- 사용자가 직접 입력해야 하는 비밀번호 자동 입력.
- 영어 정렬 라벨 `Hot`, `New`, `Top`, `Rising`.
- 과한 히어로 섹션.

접속 상태 기준:

- 2026-07-09 확인 기준 `https://bachata.co.kr`은 GitHub Pages에서 200 응답.
- GitHub Pages HTTPS 인증서는 approved.
- `www.bachata.co.kr`은 apex로 리다이렉트.
- Worker URL도 200 응답.

주의:

- GitHub Pages CDN 캐시는 최대 600초 정도 남을 수 있다.
- 정적 HTML과 Worker API가 분리되어 있으므로 캐시 갱신 시점 차이가 생길 수 있다.
- 실제 동적 커뮤니티 기능은 Worker API가 살아 있어야 한다.
